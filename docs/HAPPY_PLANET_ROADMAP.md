# 快乐星球 · 档案与任务玩法扩展方案（Roadmap）

> ✅ **P1 + P2 已落地**（羁绊阶段 / 任务连击 / 徽章 / 纪念日仪式 / 星球周报+分享卡 / 能量曲线 / 记忆星图）。
> 交付说明与逐个演示脚本见 [HAPPY_PLANET_GROWTH.md](HAPPY_PLANET_GROWTH.md)；迁移为 `V14__happy_planet_growth.sql`。
> 下方 P3/P4（主题剧情季 / 观察日志 / 家庭星球 / 订阅）仍为规划。


在 Demo 已交付能力（能量 / 档案 / 频道 / 每日任务 / 首连仪式 / App 端）基础上，围绕两条主线扩展：
**档案 → 关系资产**（让用户越用越难离开），**任务 → 内容运营引擎**（让内容可持续、可审核、可付费）。
每个功能都标注了复用的现有设施，保证可落地。

---

## 一、设计原则（也是对投资人讲的故事）

1. **每日仪式钩子**：晚安频道 + 每日任务 + 连击，构成"每天必须打开一次"的习惯回路 → 拉 DAU/DAU 比。
2. **沉没成本随时间累积**：档案、观察日志、羁绊等级都是"搬不走的记忆" → 切换成本即留存。
3. **内容全部逐字可审**：所有新增玩法的台词继续走 `sys_script_lines`（不经过 LLM），这是对 IP 方（甲方）的核心差异化卖点——"内容飞轮转得快，但品牌方向盘在甲方手里"。
4. **每个玩法自带可量化指标**：连击 → D7 留存；周报分享 → K 因子；档案深度 → 流失预测。投资人看得到数据抓手。

---

## 二、方向一：档案 → 关系资产

### 1. 羁绊阶段（关系等级）⭐ 优先级最高
- **玩法**：陪伴天数 + 累计互动 → 关系阶段：`初识 → 熟悉 → 老朋友 → 星球密友`。升级时设备主动播报升级仪式（惊喜时刻），每个阶段解锁新台词包、新档案字段、新频道能力。
- **粘性逻辑**：升级期待感 + 阶段专属内容，把"陪伴天数"从一个数字变成可感知的进度条。
- **落地**：阶段由 `AgentRuntimeService` 根据 `sys_agent_state.companionDays` 计算（无需新表）；升级仪式台词走 `sys_script_lines`（sceneKey=`stage_up_2/3/4`），播报复用 `HappyPlanetSpeaker`；阶段写入动态状态提示词影响语气亲密度。
- **投资人视角**：关系等级分布 = 天然的用户生命周期漏斗图。

### 2. 记忆星图（档案可视化 + 分层解锁）
- **玩法**：App 端把档案渲染成一张星图，每个已记录的档案字段 = 一颗点亮的星。档案白名单按羁绊阶段分层扩容（如阶段 2 解锁 `favoriteFood/hobby`，阶段 3 解锁 `recentGoal/comfortTopic`），星图上灰色星星提示"还有什么没被记住"。
- **粘性逻辑**：收集欲 + "AI 越来越懂我"的可视化证据；灰色星星是天然的对话引导。
- **落地**：`AgentProfileService` 白名单改为按阶段分层的 Map；App 端在 `PlanetAppView.vue` 加星图组件；数据接口 `/api/app/planet` 已有。
- **边界**：分层白名单仍是代码级硬约束，敏感信息照旧拒收，安全故事不变。

### 3. 星球周报 + 纪念日引擎 ⭐ 裂变入口
- **玩法**：每周日晚生成"本周星球报告"：快乐能量曲线、完成任务数、连击、本周新记住的 2~3 件事、一句专属寄语；App 内以精美卡片呈现，一键生成分享图（带邀请码）。`importantDate`、陪伴第 7/30/100 天自动触发设备端纪念仪式播报。
- **粘性逻辑**：周报是"被看见"的情感反馈；纪念日是无成本的惊喜时刻。
- **落地**：复用 `HappyPlanetTaskScheduler` 加每周任务；能量曲线需要新表 `sys_energy_log`（`EnergyService` 每次调整能量时追加一行事件，改动约 10 行）；纪念仪式台词走 `sys_script_lines`；分享图前端生成（canvas）。
- **投资人视角**：分享卡片 = 零成本获客渠道，可直接算 K 因子；能量曲线数据本身是"情绪陪伴有效性"的证据资产。

## 三、方向二：任务 → 内容运营引擎

### 4. 任务连击（Streak）+ 星球徽章 ⭐ 优先级最高
- **玩法**：连续完成每日任务累计连击；3/7/21 天达成时授予徽章（"地球观察员""能量守护者"），设备播报授勋仪式，App 徽章墙展示。断签一天不清零，用"能量修复卡"（每月 1 张）补签——温和而非焦虑的连击设计，符合陪伴调性。
- **粘性逻辑**：Streak 是被验证过的最强日活钩子；补签卡避免"断签即弃坑"。
- **落地**：`sys_agent_state` 加 `streakDays/streakRepairLeft` 两列；`CompleteTaskFunction` 里加连击结算；徽章新表 `sys_agent_badge`（deviceId, roleId, badgeKey, earnTime）；授勋台词走 `sys_script_lines`。
- **投资人视角**：连击分布直接对应 D7/D30 留存曲线，是路演最好讲的数字。

### 5. 主题任务周 / 剧情季（内容飞轮）
- **玩法**：任务从"每日随机一条"升级为"7 天一个主题链"：如"地球观察周"（周一找一朵云 → 周日把这一周的发现讲给我听），完成整周获得主题勋章 + 解锁一段星球剧情语音。按季度上新剧情季（"信号丢失的第七天"等轻科幻短剧情），旧季归档可回放。
- **粘性逻辑**：从"打卡"升级为"追更"，任务有了叙事悬念；季更节奏 = 持续的回流理由。
- **落地**：`sys_planet_task` 加 `chainKey/chainSeq` 两列，`PlanetTaskService.randomTask` 改为"有进行中的链则取下一条，否则开新链"；剧情台词继续走 `sys_script_lines` 逐字审核——**IP 方全程可控，这是对甲方续约和 IP 授权谈判的关键筹码**。
- **投资人视角**：内容成本极低（一季 = 数十条审核台词 + 任务文案），但形成了"内容运营 → 回流 → 数据 → 下一季选题"的飞轮；也是天然的付费点（见方向三）。

### 6. 地球观察日志（任务回音，UGC 沉淀）
- **玩法**：完成任务时智能体追问一句"跟我说说你发现了什么？"，把用户的回答摘要成一条"观察记录"存档；App 端形成时间线"我的地球观察日志"，周报和剧情季会引用这些记录（"上周你说天空是橘色的"）。
- **粘性逻辑**：用户自己的话被记住、被引用，是最强的情感锁定；日志时间线是不可迁移的个人资产。
- **落地**：新表 `sys_task_log`（deviceId, roleId, taskId, userEcho, createTime）；`CompleteTaskFunction` 加可选参数 `echo`，由 LLM 在对话中自然收集；动态状态提示词注入最近 1~2 条记录实现"引用"。
- **边界**：记录前口头确认"要我记下来吗"，管理端/App 可删，延续"透明可控"故事。

### 7. 家庭星球（双端协同任务）
- **玩法**：family 类任务升级为双端确认：设备端发起"和家人互相说谢谢"，家人在 App 上点亮确认后任务才完成，双方能量同时补充；家庭成员共享一个"家庭能量池"，月度生成家庭报告。
- **粘性逻辑**：一台设备激活多个 App 账号（长辈用设备、子女用 App），社交约束提升完成率，天然口碑传播。
- **落地**：`sys_player_agent` 已支持多用户关联同一设备；加确认接口到 `MobileAppController`；家庭能量池在 `sys_agent_state` 维度上聚合即可。
- **投资人视角**：单设备多账号 = ARPU 放大器；"银发陪伴 + 子女远程关心"是当前融资环境里最好讲的赛道故事。

## 四、方向三：商业化钩子（值得落地的原因）

| 付费点 | 依托玩法 | 形态 |
|---|---|---|
| 星球会员订阅 | 剧情季提前解锁、周报深度版（月度情绪陪伴报告）、连击补签卡加量 | 月订阅 |
| 限定语音包 | 剧情季角色声线、节日限定台词包（`sys_script_lines` + TTS 音色，全部可审） | 单次购买 |
| IP 联名 | 快乐星球正版剧情季、联名硬件皮肤 | B 端授权分成 |
| 家庭套餐 | 家庭星球多账号 + 家庭报告 | 家庭订阅 |

数据资产故事：`sys_energy_log` 累积的匿名化情绪曲线 + 任务完成行为，是"AI 情感陪伴有效性"的量化证据，支撑后续 B2B2C（康养机构、保险、运营商礼品机）叙事。

## 五、落地节奏建议

| 阶段 | 内容 | 主要改动 | 预期指标 |
|---|---|---|---|
| P1（约 2 周） | 任务连击+徽章、羁绊阶段、纪念日仪式 | state 表加列、badge 新表、台词种子数据、FunctionCall 微调 | 次日/7 日留存 |
| P2（约 3 周） | 星球周报+分享卡、能量曲线、记忆星图 | `sys_energy_log` 新表、周报调度、App 两个新页面 | K 因子、周活 |
| P3（约 4 周） | 主题任务周/剧情季、观察日志 | task 表加链、`sys_task_log`、内容生产流程 | 30 日留存、内容回流率 |
| P4 | 家庭星球、订阅体系 | 双端确认接口、支付接入 | ARPU、账号/设备比 |

P1 全部是现有表加列 + 台词种子 + 少量服务逻辑，无架构改动，适合先跑给甲方和投资人看闭环。
